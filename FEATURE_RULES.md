# Feature Rules

## Anonymous Posts

Anonymous posts must:
- hide username
- hide profile picture
- use ghost avatar
- avoid linking to real profile
- show label "Anonim"

Anonymous posts must not expose:
- real user identity
- user profile route
- identifiable avatar

---

## Mood System

Mood is required before posting.

Mood badge must appear:
- in feed
- in post detail
- in user post history

Mood badge may affect:
- badge color
- small accent decoration
- filter category

Mood must not make the post visually unreadable.

---

## Feed Rules

Default feed behavior:
- chronological order
- simple post cards
- no infinite scroll during early MVP unless needed
- use dummy data in Phase 1

Feed should avoid:
- clutter
- too many buttons
- aggressive recommendation system
- unnecessary viral metrics

---

## Comment Rules

Comments should feel supportive.

Comment UI must:
- be simple
- show avatar
- show username
- show time
- show comment text

Avoid:
- toxic copywriting
- ranking comments by popularity during MVP
- complex nested replies during MVP

---

## Like Rules

Like action should:
- feel soft
- use subtle animation
- update optimistically in dummy state

Do not use confetti.

---

## Bookmark Rules

Bookmark action should:
- toggle visually
- use clear icon state
- not require backend during Phase 1

---

## Theme Rules

Theme changes should:
- preview live
- persist locally during MVP
- not break readability
- apply to accent buttons
- apply to selected mood/card accents where appropriate

Themes:
- Purple
- Pink
- Blue
- Green
- Orange

Card styles:
- Minimal
- Lucu
- Lembut
- Neon

---

## Notification Rules

Group notifications by:
- likes
- comments
- follows

Notification copy should be:
- Indonesian
- warm
- short
- clear

Examples:
- "Raka menyukai curhatmu"
- "Mira mengomentari ceritamu"
- "Dina mulai mengikutimu"

---

## Empty States

Every empty state must:
- feel friendly
- include mascot if appropriate
- include soft encouragement text

Examples:
- "Belum ada curhat di sini."
- "Ceritamu bisa jadi yang pertama."
- "Tempat ini masih sepi, tapi aman."

---

## Loading States

Loading UI should:
- feel lightweight
- avoid skeleton overload
- use subtle animation
- not distract the user

---

## Explore Rules

Explore should support:
- mood filters
- trending curhats
- search input

During MVP:
- use dummy trending data
- use local filtering

---

## Profile Rules

Profile should include:
- avatar
- username
- bio
- stats
- user posts
- theme settings entry

Stats:
- posts
- followers
- following

During MVP:
- use dummy user data

---

## Settings Rules

Settings should include:
- theme settings
- privacy settings placeholder
- notification settings placeholder
- account settings placeholder

During MVP:
- UI only
